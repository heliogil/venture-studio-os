# Kill Criteria

Every venture has a written kill condition before Sprint 1 starts. This removes emotion from the decision and prevents the sunk-cost trap.

## The formula

```
KILL if:  cost is ongoing  AND  no active signal
```

A venture is archived when it is consuming resources (factory time, infrastructure cost, founder attention) without producing a signal that justifies continuation.

**What counts as an active signal:**
- Revenue (any)
- Waitlist growth > 0
- Organic inbound (someone found us without paid ads)
- A partnership conversation in progress
- A metric trending in the right direction week-over-week

**What does NOT count:**
- "We're almost ready to launch"
- "The product is good, we just need more marketing"
- "We haven't really tried yet"

## The 90-day gate

Every venture gets a 90-day window post-launch. At the gate, we score against the metrics defined in Sprint 0.

If the gate metrics are not met, the default is **archive** — not "give it more time."

More time is only justified by a **specific, testable hypothesis** about what changes in the next 30 days and why.

## What archive means

Archive is not failure. It means:

1. All code, data, and learnings move to the vault
2. The data assets (price histories, scraped datasets, ML models) are tagged as reusable
3. A post-mortem packet is sent to Caliope — what we learned, what would need to be true for this to be revisited
4. The factory cycles free up for the next experiment

The cost of a failed experiment is 6–8 weeks of factory time. The cost of a zombie venture is indefinite.

## Practical application

```
FPSReal kill criteria:
  90 days post-launch:
  - < 500 affiliate clicks → archive
  - 0 media coverage (creator 100k+) → review, not automatic archive
  - Both → archive immediately

Zebra Shield kill criteria:
  90 days post-launch:
  - < 50 paid subscribers → archive
  - MRR < €300 → review hypothesis
  - Churn > 30%/month → pivot pricing or archive
```

These are written before Sprint 1. They do not change because "we need more time."
